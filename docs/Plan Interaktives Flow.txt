Plan: Interaktives Flow-Diagramm im Canvas (farbige Strahlen + "Flow"-Tab)

 Kontext

 Der Nutzer möchte ein interaktives Live-Diagramm im Canvas, das zeigt wie Agenten miteinander kommunizieren.
 Wanderndes Licht entlang der Delegationspfade: gelb = aktiv, grün = erfolgreich, rot = Fehler.
 Zusätzlich soll ein neuer "Flow"-Tab das gesamte Timus-System hierarchisch zeigen (Agenten + Autonomie-Motoren).

 Was bereits existiert (keine Reimplementierung nötig):
 - animateDelegationBeam(fromId, toId) — Canvas-Overlay-Animation, 700ms, Linie 2125 in canvas_ui.py
 - updateGraphNodeColor(agent, status) — Knotenfarbe via SSE, Linie 2053
 - handleSSE(d) — SSE-Dispatcher, Linie 1571; d.type === "delegation" → ruft bereits animateDelegationBeam(d.from, d.to) auf
 - _delegation_sse_hook in server/mcp_server.py Linie 656: sendet {type:"delegation", from, to, **status**} — Status-Feld ist bereits in der SSE-Payload vorhanden
 - Tab-System: .tab-btn / .tab-content / switchTab() — einfach erweiterbar
 - Cytoscape.js via CDN bereits geladen

 Was fehlt:
 1. _delegation_sse_hook wird nur für "running" gerufen (agent_registry.py Linie 306) — nicht für "completed"/"error"
 2. animateDelegationBeam() ignoriert d.status — alles goldgelb
 3. Neuer "Flow"-Tab mit vollständiger Systemhierarchie existiert noch nicht

 Schicht 0: [Telegram] [Terminal]

            [visual] [data] [communication] [system] [shell] [image]
 Schicht 4 (rechts, eigene Spalte): [AutonomousRunner]

                     [M14 Email]  [M15 Ambient] [M16 Feedback]

 Nodes als flowCy (zweites Cytoscape-Objekt, selector #flow-cy).
 Layout: {name: "dagre", rankDir: "TB", nodeSep: 40, rankSep: 70}.
 flowCy erhält die gleichen Node-Style-Regeln wie cy.

 animateFlowBeam(fromId, toId, status) — identische Logik wie animateDelegationBeam,
 aber auf #flow-beam-overlay Canvas + flowCy-Positionen.
 Eigener _flowBeamRAF für unabhängige Animation.
