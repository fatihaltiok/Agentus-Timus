 Kontext

 Timus hat 27 Lean-Theoreme die mathematisch Invarianten beweisen — aber bisher ohne direkte Verbindung zum Python-Code. Lean beweist die Invariante isoliert; Python könnte sie trotzdem
 verletzen. Ziel: Diese Lücke schließen durch (1) Hypothesis property-based Tests die aktiv Gegenbeispiele suchen, und (2) CrossHair-Contracts direkt in Python-Funktionen.

 Was bereits existiert (wird genutzt):
 - hypothesis in requirements.txt ✅, .hypothesis/ Verzeichnis vorhanden
 - tests/test_hypothesis_algorithms.py — 180 Zeilen, bewährtes Pattern (@given, @settings, assume)
 - Vollständige Type Hints in M14/M13/M16 ✅
 - 27 Lean-Theoreme in lean/CiSpecs.lean (alle by omega) ✅
 - Python 3.11.13, pytest asyncio_mode=auto, conftest.py mit Integration-Flag

 Warum CrossHair statt Nagini:
 Nagini unterstützt offiziell max. Python 3.10. CrossHair unterstützt Python 3.8–3.12 ✅, symbolische Ausführung via Docstring-Contracts, pip install crosshair-tool ohne Java/Viper.

 Phase 2 — CrossHair: Pre/Postconditions in Python-Funktionen

 Installation: pip install crosshair-tool + in requirements.txt eintragen.

 CrossHair liest Contracts aus dem Docstring (PEP 316-Style) und verifiziert sie symbolisch.

 orchestration/email_autonomy_engine.py

 def _in_whitelist(self, recipient: str) -> bool:
     '''
     post: not __return__ or len(self.recipient_whitelist) > 0
     post: __return__ == any(r.lower() == recipient.lower() for r in self.recipient_whitelist)
     '''

 def evaluate(self, context, recipient, subject, body, confidence=0.9) -> EmailDecision:
     '''
     post: 0.0 <= __return__.confidence <= 1.0
     post: not __return__.should_send or self._in_whitelist(__return__.recipient)
     post: not (__return__.confidence < self.confidence_threshold) or not __return__.should_send
     '''

 orchestration/tool_generator_engine.py

 def validate_ast(self, code: str) -> tuple:
     '''
     post: not (len(code) > self.MAX_CODE_LENGTH) or not __return__[0]
     post: isinstance(__return__[0], bool) and isinstance(__return__[1], str)
     '''

 def activate(self, action_id: str) -> bool:
     '''
     pre: action_id in self._tools
     post: not __return__ or self._tools[action_id].status in ("active",)
     '''

 orchestration/feedback_engine.py

 def get_hook_stats(self, hook_name: str) -> dict:
     '''
     post: 0.05 <= __return__["weight"] <= 2.0
     post: __return__["total"] == __return__["pos"] + __return__["neg"] + __return__["neutral"]
     '''
